# Tasks: Embedded Vector Store Implementation

**Agent Instruction:** Implement the host-embedded vector store using a native Rust HNSW algorithm and expose it to WebAssembly guests via Wasm Interface Types (WIT).

- [ ] **Interface Definition:** Create `wit/store/vector.wit` exposing `upsert`, `search`, and `delete` methods.
- [ ] **Rust HNSW Engine:** Integrate or develop an HNSW index within `embedded-core-store` capable of asynchronously persisting vector graphs to NVMe disk.
- [ ] **Host Bindings:** Link calls generated by `wasmtime-bindgen` to vector store operations, ensuring isolation context (Tenant ID from `system-faas-authz`) strictly partitions index access.
- [ ] **Optional TDE (Security):** Ensure disk storage for these indices passes through the encryption-at-rest layer (AES-GCM) specified in Phase 3 if the FaaS module requires it.
- [ ] **Guest Example:** Update (or create) a project in `examples/` demonstrating a Rust FaaS call that vectorizes text and performs a similarity search.
