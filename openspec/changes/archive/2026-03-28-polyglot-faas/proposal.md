# Proposal: Change 011 - Polyglot WASM FaaS (Go & JavaScript)

## Why
Tachyon Mesh already demonstrates Rust-to-WASM execution, but it does not yet prove that the host runtime is language-agnostic. To validate the routing and execution model, the platform must execute guest modules produced by non-Rust toolchains without introducing language-specific branches in the host.

## What Changes
- Add a `guest-go` example compiled to WASI with TinyGo.
- Add a `guest-js` example compiled to WASI with Javy.
- Extend `core-host` so it can invoke either the existing `faas_entry` export or the standard WASI `_start` export.
- Update the container build and sealed `integrity.lock` routes so both new guests ship with the runtime image.
- Extend the k3d integration workflow to assert the new `/api/guest-go` and `/api/guest-js` responses.

## Success Metrics
- `GET /api/guest-go` returns a response generated by the Go guest.
- `GET /api/guest-js` returns a response generated by the JavaScript guest.
- The shared Wasmtime/WASI execution path continues to serve the existing Rust guests unchanged.
