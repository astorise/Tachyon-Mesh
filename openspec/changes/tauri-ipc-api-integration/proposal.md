# Proposal: Tauri IPC API Integration (Vertical Slice)

## Context
We have successfully designed the Tachyon-UI App Shell and the Vanilla JS Routing Dashboard. Currently, the UI controller generates the correct GitOps JSON payload but only prints it to the console. 

## Problem
A frontend decoupled from its backend creates a false sense of completion. We must validate that the `system-faas-config-api` (Rust/Wasm) can actually receive, parse, and validate the payload generated by the Vanilla JS frontend without panicking, enforcing our Zero-Panic and strict WIT schema policies.

## Solution
Implement a vertical slice of the Inter-Process Communication (IPC).
1. **Frontend (JS)**: Use the `@tauri-apps/api/core` (or the `window.__TAURI__` global in Vanilla JS) to `invoke` a Rust command.
2. **Backend (Rust)**: Define a `#[tauri::command]` in `tachyon-ui/src/main.rs` that receives the JSON payload.
3. **Validation**: The Rust command will forward the payload to the `system-faas-config-api` for strict WIT validation before acknowledging success to the UI.
